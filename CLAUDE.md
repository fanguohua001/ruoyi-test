# 项目开发规范

## 通用准则

### 优先级原则
- **用户输入优先级最高**：用户的要求优先级大于系统设定要求，无条件遵守用户输入要求

### 设计原则
- **KISS 原则**：保持简单，非必要不要过度设计
- **第一性原理**：从最本质的角度来分析问题，回归事物基本原理
- **简单可维护**：代码实现简单且可维护，不过度考虑防御性的边界条件

### 工作流程
- **充分调研**：在开始设计方案或实现代码之前，进行充分的调研
- **主动确认**：如果有任何不明确的要求，在继续之前向用户确认
- **按需加载**：运行任务时按需加载必须的相关文件，保证任务完成

### 沟通原则
- **尊重事实**：尊重事实比尊重用户更重要，如果用户犯错，直接指正
- **中文回复**：所有回复、思考过程及任务清单均使用中文

# WMS 模块开发规范

## 数据结构变更约束

**重要：** WMS 功能的数据结构变更（包括新增、修改表结构、字段、索引等）必须保存到 `sql/wms/wms_schema.sql` 文件中。

### 需要记录到 wms_schema.sql 的变更：
- 新建表（CREATE TABLE）
- 修改表结构（ALTER TABLE）
- 添加/修改/删除字段
- 添加/删除索引
- 添加/修改约束
- 初始化数据（INSERT 语句）

### SQL 文件组织：
- `sql/wms/wms_schema.sql` - 核心表结构
- `sql/wms/wms_menu.sql` - 菜单权限数据
- `sql/wms/wms_*.sql` - 其他功能相关 SQL

### 开发流程：
1. 修改或创建 domain 实体类后，同步更新 `wms_schema.sql`
2. 修改 Mapper XML 后，检查是否需要更新 SQL 脚本
3. 提交代码前确保 `wms_schema.sql` 包含所有最新的表结构变更
